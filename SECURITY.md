# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in django-smart-layer, **please do not open a public GitHub issue.** Public disclosure before a fix puts users at risk.

Instead, report it privately via GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) ("Report a vulnerability" under the repository's Security tab), or by emailing **ayushiagrawal2507@gmail.com** directly.

Please include:

* A description of the vulnerability and its impact
* Steps to reproduce (or a proof of concept)
* The affected component (middleware class, model, or utility) and file/route if known

We'll acknowledge your report within 72 hours and keep you updated on the fix. Once resolved, a patched version will be released and you will be credited in the changelog unless you prefer otherwise.
